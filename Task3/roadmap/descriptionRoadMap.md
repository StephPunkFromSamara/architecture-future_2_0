# Соответствие шагов, мер и роадмапа

## Шаги, меры и их связь с роадмапом

## Формирование арх.решения
### Применение микросервисной архитектуры
- **Мера:** Разделение монолита на микросервисы для облачной миграции.
- **Связь с роадмапом:**
    - **Q1:** Формирование бэклога и прототипов архитектурных решений 
    - **Q2:** MVP сервисов и параллельное развертывание в облаке 
    - **Команды:** Архитектурная команда, продуктовые команды, DevOps
    - 
### Безопасность данных (PrivacyByDesign)
- **Мера:** Шифрование данных, безопасные протоколы передачи (HTTPS, TLS), а также контроль доступа.
- **Связь с роадмапом:**
    - **Q1:** Исследование готовности сервисов к миграции в облако 
    - **Q2:** Развертывание и настройка сервисов с безопасностью 
    - **Q3:** Тестирование ключевых сервисов с соблюдением безопасности (ML, DLH, Финтех, CRM)
    - **Команды:** Команда инноваций и ИБ, инфраструктурная команда, облачный security

## Исследование готовности сервисов к миграции в облако и инфраструктуры/Развертывание и настройка и миграция сервисов по готовности
### Безопасность на стороне облачного провайдера
- **Мера:** Настройка ролей, прав доступа, аудит, мониторинг.Использование IaaS и private cloud для отдельных сервисов.
- **Связь с роадмапом:**
    - **Q2:** Параллельное развертывание в облаке с инструментами миграции 
    - **Q3:** Тестовый запуск ключевых сервисов
    - **Команды:** Инфраструктурная команда, облачный security
### Настройка инфраструктуры и интеграций
- **Мера:** Настройка брокера сообщений, S3-хранилища, Iceberg; написание интеграционного кода.
- **Связь с роадмапом:**
    - **Q1:** Работающие прототипы решений 
    - **Q2:** Развертывание и настройка сервисов 
    - **Команды:** DevOps, Data Engineers, продуктовые команды

## Исполнение продуктового бэклога

### Обезличивание данных для Data Lake House
- **Мера:** Псевдонимизация и анонимизация PII перед загрузкой в DLH.
- **Связь с роадмапом:**
    - **Q2:** Настройка DLH и интеграция с аналитикой/ML 
    - **Q3:** Тестовый запуск DLH с безопасными данными
    - **Команды:** Data Engineers, продуктовые команды

### Формирование бэклога по переходу с DWH на DLH
- **Мера:** Определение приоритетов миграции, задач и этапов.
- **Связь с роадмапом:**
    - **Q1:** Формирование бэклога 
    - **Q2:** Исполнение продуктового бэклога, реализация новых интеграций и подготовка сервисов для контейнеризации
    - **Q3–Q4:** Тестирование сервисов, обучение пользователей, A/B тестирование 
    - **Команды:** Архитектурная команда, продуктовые команды, инфраструктурная команда


## Ожидаемые результаты по кварталам

| Квартал | Основные достижения |
|---------|------------------|
| Q1 | Формирование бэклога, работающие прототипы решений |

| Q3 | Тестовый запуск ключевых сервисов в облаке (ML, DLH, Финтех, CRM) |
